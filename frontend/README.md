# CadArena Frontend

The frontend is a React 18 application with an embedded static Studio workspace. It provides public pages, authenticated application routes, profile and community screens, a DXF viewer, and a RAG chat interface.

## Runtime Parts

| Part | Location | Responsibility |
| --- | --- | --- |
| React application | `src/` | Main routing, layouts, pages, authentication context, UI components, API clients, and viewer screens. |
| Embedded Studio | `studio-source/` | Static CAD workspace source for projects, prompt chat, generation, iteration, preview, and DXF actions. |
| Copied Studio assets | `public/studio-app/` | Generated copy served by the React dev server and backend static server. |
| Copy script | `scripts/copy-studio.js` | Copies `studio-source` into `public/studio-app` before `npm start` and `npm run build`. |
| API service | `src/services/api.js` | Backend and RAG Axios clients, health checks, RAG calls, DXF actions, and application API helpers. |
| Auth API service | `src/services/authApi.js` | Login, logout, registration, current user, profile updates, and avatar operations. |

## Routes

| Route | Access | Component |
| --- | --- | --- |
| `/` | Public | Home page |
| `/about` | Public | About page |
| `/developers` | Public | Developers page |
| `/docs` | Public | Documentation page |
| `/login` | Public | Login page |
| `/signup` | Public | Sign-up page |
| `/community` | Authenticated | Engineering community |
| `/generate` | Authenticated | Generation workspace page |
| `/models` | Authenticated | Model information page |
| `/metrics` | Authenticated | Runtime metrics page |
| `/profile` | Authenticated | Profile page |
| `/profile/edit` | Authenticated | Profile editing page |
| `/studio` | Authenticated | Embedded Studio workspace |
| `/viewer` | Authenticated | DXF viewer |
| `/rag-chat` | Authenticated | RAG-backed architecture chat |

Authentication state is provided by `src/contexts/AuthContext.js`. Protected routes redirect unauthenticated users to `/login?from=<path>`.

## Environment

Copy the example file:

```bash
cp .env.example .env.local
```

Common values:

```bash
REACT_APP_API_URL=http://localhost:8000
REACT_APP_RAG_API_URL=http://localhost:8001
REACT_APP_API_TIMEOUT=180000
REACT_APP_ENABLE_AUTH=true
REACT_APP_ENV=development
GENERATE_SOURCEMAP=false
```

When `REACT_APP_API_URL` is not set and the app runs on port `3000`, the API client falls back to `http://localhost:8000`. The RAG client falls back to `http://localhost:8001`.

## Local Development

```bash
cd frontend
npm install
npm start
```

`npm start` runs `prestart`, which copies the Studio source into `public/studio-app`.

Open:

- `http://localhost:3000/`
- `http://localhost:3000/studio`
- `http://localhost:3000/community`
- `http://localhost:3000/rag-chat`

## Build

```bash
npm run build
```

`npm run build` runs `prebuild`, copies the Studio source, and then creates the React production build under `build/`. The backend serves this build when `frontend/build/index.html` exists.

## Studio Workflow

- Edit Studio source files in `studio-source/`.
- Do not edit `public/studio-app/` directly; it is generated by `copy-studio.js`.
- Run `npm start` or `npm run build` to refresh the copied Studio files.

## Verification

Run tests:

```bash
npm test
```

Run a production build:

```bash
npm run build
```

The frontend expects the backend to be reachable for authenticated pages, workspace generation, community operations, profile operations, DXF actions, and ArchChat. The RAG chat page also expects the standalone RAG API when retrieval is used.
